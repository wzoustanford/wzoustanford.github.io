What stage are you?

We have built the Reinforcement Learning Models for US equity, and it has been put into the production system for 2 months. The system takes in public exchange data (price, volumne), news, macro-economic indicators, and social network information to understand the impact of action taking in the financial market.

Without a deep Reinforcement Learning Model, we found a generative LLM model will be more likely to halucinate or dabble about investing, because it does not take into account of penalties and rewards of taking real market actions. In an extremely noisy environment like financial news, markets, even at the company reports level, it is our view a foundational Reinforcement Learning model is the way to go in the next 5-10 years for financial intelligence.

We've created a parsed data-base of all SEC filings, and are fine-tuning and aligning an genAI LLM model. This model is planned to be integrated with forecasting and RL models for global financial intelligence.

For the Angle application, we’ve built a vertical application with UX, graphs, widgets, data-bases, search, crawlers. We are now building the AI horizontal infra. The first version of the product consists of two LLM scoring play-grounds, one for UX/front end, for reasoning LLMs to author UX interfaces on the fly either in conversation or customize what the user sees. The second play-ground is used to teach LLMs to reason and call microservice interfaces, such as get data, search, store data, parse and interpret. Put together, a vertical application is built and highly customizable by the user through the AI system. Our approach is build a ‘lego world’ for AI. This ‘lego world’ is composed of front end user UX and distributed data systems. In both worlds, we work with excellent engineers to design reliable, versatile, effective lego blocks (widgets) for AI to understand, manipulate, and customize.

Why do we build an ML infra for Financial RL?

[High calibre decisions and statistical significance] The ML iterations are more frequent and more intense for financial data. This is because the impact of each decision carry a high value, and a less competent model or a even a less interpretable model is constantly under scrutiny. The second reason is statistical significance: more iterations are needed for finance because the market environment is noisy filled with heavy-tail events. To reach statistical significance in this environment requires many more iterations on the GPU ML platform.

[Prime transactional platform for reliable production, on-line experiments, and back-tests] Financial data requires reliable, fast, and well-documented interfaces for transactions. This is the foundation for production and online experiments. The GPU serving of the RL model is required to be efficient, available, and developer friendly. Despite the wide interest in webservices and ML, financial transactions and ML infra is a black-box and secretive, usually custom built and maintained by hedge funds or banks. We build this infra to validate, serve and put our Foundational RL mode into production. We also make this available for ML practitioners at investor serving institutions and enterprises.

[Engineering, data, algorithm excellence means a world of difference] We build an ambitious goal for the first foundational RL model for finance, while the accuracy to each second of financial data and each financial action to take is important. So engineering excellence, efficient algorithms and low-level data design means a word of difference to capture signals. Our ML infra is designed from the first line of code, till this date, each line was implemented, reviewed by the founders.

Sophon

Sophon is the agent that determines what users see. We use Sophon to discover, show interactive interfaces, run programs, and adapt to user's preference.

For a demo of Sophon on mobile, we show a financial chat system. The user experience is defined by generative AI. When user inputs the prompt, the generative AI interprets the request, and calls relevant UX, backend components to offer a comprehensive response. The interface is customized to and steerable by the user. In this example, a interactive stock chart was loaded, a browser was loaded to look for information for CEO's, and a pdf viewer together with back-end database was called to display financial reports of the companies asked. Sophon is a system platform that allows services, systems, interfaces to be operable and steerable.

Weaver

Weaver is the agent that weaves a web of information and connects the users questions to the most up-to-date information. We use Weaver to find the user-customized, the most suitable, and the most updated information source. Weaver is ensured to act very fast, and is a nimble superhero.Try weaver with your own prompt (this demo is for asking financial/stock questions):
